# 使用说明

本文档对应当前版本 `1.2.85`，面向日常使用和 VPS 运维操作。

## 1. 启动

系统只通过 VPS Docker 服务运行。先在 VPS 上进入项目目录：

```bash
cd procure-lite
cp .env.example .env
docker compose up -d --build
```

默认端口是 `8000`。如需调整，修改 `.env` 中的 `PROCURE_LITE_PORT`：

```bash
PROCURE_LITE_PORT=8000
PROCURE_LITE_AUTH_COOKIE_SECURE=auto
```

启动后访问：

```text
http://VPS公网IP:8000
```

如果已经配置域名和 HTTPS，则访问：

```text
https://你的域名
```

更多步骤见 [VPS 部署教程](./docs/vps-deployment.md)。

## 2. 首次使用

首次进入系统时，需要设置管理员密码。请妥善保存恢复码。

之后使用管理员密码登录。系统会使用本地 Cookie 会话保持登录状态。

## 3. 界面导航与首页

登录后会进入控制台式首页。左侧导航用于切换主要业务区域，顶部栏显示当前页面、全局搜索、今日日期、待办提醒和管理员入口。

首页主要包含：

- KPI 卡片：集中显示待采购、待到货、待分发和待报销数量，点击卡片可进入对应台账筛选或运营处理页
- 采购台账预览：展示最近采购记录的流水号、部门、经办人、物品、数量、状态和供应商，点击记录可查看详情和审计轨迹
- 今日待办：展示快速下单、确认收货、报销跟进等行动队列，可直接进入运营工作台处理
- 状态分布和采购周期：用于快速观察当前采购闭环压力，详细分析仍以统计报表页为准

顶部搜索框支持按物品、单据或供应商等关键词搜索。输入关键词后按回车或点击 `搜索`，系统会自动切换到台账页并按关键词筛选。

运营工作台和统计报表采用顶部二级标签切换，例如 `总览`、`采购跟进`、`主数据`、`异常处理`。左侧导航只负责切换业务区域，具体子页面在顶部标签中切换。

## 4. 导入单据

1. 点击 `导入单据` 或概览页的 `上传采购单据`。
2. 上传 `PDF / PNG / JPG / JPEG / JFIF` 文件。
3. 系统创建后台解析任务，前端自动轮询任务状态。
4. 解析完成后检查预览结果。
5. 根据需要校正流水号、部门、经办人、申领日期、物品名称、数量和采购链接。
6. 确认导入。

解析默认使用本地 PaddleOCR：

- 无需配置 API Key
- 单据文件不会上传到云端解析服务
- PDF 优先使用文本/表格解析，必要时使用 PaddleOCR 兜底
- 图片文件直接使用 PaddleOCR 解析

## 5. 台账

台账支持：

- 按关键词、采购状态、部门、月份筛选
- 分页浏览，支持每页 20/50/100 条
- 在线编辑流水号、部门、经办人、物品名称、数量、单价、状态、日期和备注
- 批量修改状态或付款状态
- 批量删除到回收站

手机窄屏会自动切换为卡片视图，便于现场查询和更新状态。

## 6. 执行看板

执行看板固定三列：

- 待采购
- 待到货
- 待分发

可拖拽卡片跨列流转，也可以使用“一键流转”。到货和分发阶段可填写日期与签收备注。

## 7. 运营工作台

运营工作台面向日常采购跟进，保持单用户使用方式，不引入多用户权限或审批矩阵。

### 今日行动队列

总览页会按严重程度汇总待办。常见待办包括：

- 待下单：点击 `快速下单` 会用推荐供应商、今天日期和参考交期生成采购单
- 待收货：点击 `确认收货` 会填入今天日期，并推动台账进入待分发阶段
- 待报销：点击 `标记报销提交` 会把报销状态改为已提交

如果需要查看完整上下文，点击卡片或 `定位台账` 可回到对应台账记录。

### 台账闭环阶段

台账桌面表格和手机卡片都会展示闭环阶段：

- `待下单`：还需要确认供应商和采购单
- `待收货`：已下单，等待收货确认
- `待分发`：已到货，等待发放或签收
- `待报销`：已分发或已付款，但报销闭环未完成
- `已完成`：已分发且付款状态为已报销

这个阶段标签是辅助判断，不替代台账里的采购状态、发票状态和付款进度。

### 供应商与价格记忆

在 `采购跟进` 页，待下单卡片会展示推荐供应商、历史单价和参考交期。可使用：

- `套用推荐`：把推荐供应商、单价、采购链接和预计到货日期填入当前条目
- `沉淀价格记忆`：把本次单价、供应商、采购链接、流水号和交期保存为后续推荐依据

如果某个物品还没有历史价格，先填写本次单价和供应商，再沉淀为价格记忆。

## 8. 统计报表

报表页包含：

- 当前费用总计
- 已计价金额
- 单价缺失笔数
- 部门支出占比
- 状态金额占比
- 月度金额趋势
- 采购周期和执行漏斗

台账筛选条件变化后，切换到报表页会自动按最新筛选重算数据。

## 9. 数据治理

系统设置中包含：

- 回收站：恢复或彻底删除记录
- 数据巡检：扫描质量问题和重复键组
- 审计日志：查看新增、更新、删除记录，并支持部分回滚

## 10. 备份与恢复

本地备份会导出 zip，包含数据库和上传文件。系统设置页也提供自动本机备份，默认每 24 小时在 VPS 的 `procure-lite-state/backups/` 中保留最近 7 份，可在页面里立即创建或从列表恢复。

恢复时上传 zip，系统会先执行健康检查，再恢复数据。恢复期间接口可能短暂返回 `503`，稍后重试即可。

WebDAV 云同步支持：

- 保存 WebDAV 配置
- 测试连接
- 上传当前备份
- 查看远端备份并恢复

## 11. 数据落盘位置

Docker 部署默认位于：

```text
procure-lite-state/
```

迁移或备份 VPS 时，优先备份整个 `procure-lite-state/` 目录。

## 12. 常见问题

### 页面打不开

- VPS 安全组是否开放 `8000`，或域名反向代理是否已经生效
- VPS 防火墙是否允许访问该端口
- Docker 服务是否正在运行：`docker compose ps`
- 访问的是 VPS 公网 IP 或域名，不是 `127.0.0.1` 或 `localhost`

### 解析效果不稳定

- 优先使用可复制文本的 PDF
- 图片尽量保持清晰，裁掉无关区域
- 导入前在预览页校正关键字段

## 13. 更新、日志与停止

更新到最新代码并重建镜像：

```bash
git pull --ff-only
docker compose up -d --build
```

查看状态和日志：

```bash
docker compose ps
docker compose logs -f procure-lite
```

停止服务：

```bash
docker compose down
```
